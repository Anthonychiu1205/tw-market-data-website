# CLAUDE.md — TWMD 官網施工預設(工程標準)

> 完整版:`docs/ENGINEERING_STANDARDS.md`。以下 5 條鐵律 + PR gate,**每次施工都適用**。緣起:2026-07-13~14 一連串 bug(死方案表 RPM 10、捏造 2330 K線/報價、demo 假指數)其實是系統性缺口的症狀 → 治本。

## 5 條鐵律(治本,非治標)
1. **單一真相源(SSOT)、零重複、零死碼**:定價=`plans.ts`(唯一定價源)、狀態對齊 llms.txt。死碼(grep 零引用)刪不留——死碼被當活文案改過已是事故(pricingPlans/hero-market-preview)。改動只改 SSOT。
2. **零捏造對外數字**:任何 demo/placeholder 的價格/K線/指數/統計 = 地雷。**假不了就不顯示;禁假數字貼真實 ticker(如 2330)。** 官網任何對外數字走真源或整塊不顯示。
3. **CI 誠實跑全部 / 顯示走真源**:顯示的數字接 SSOT/API 真值,禁硬編碼過時值;不齊就不顯示,不編造覆蓋數字。
4. **改前先確認 render**:改/刪前 grep 確認引用;死碼(零 render)刪、活碼接真源。不改版面。
5. **改一個數字前先問「有幾個源」**:不一致 → 找 SSOT、刪其餘,根因修非症狀修(RPM 10→60→發現整代死表的教訓)。

## 每個 PR gate
- [ ] 無新重複真相源 / 無死碼(新加零引用要說明用途)
- [ ] 無捏造對外數字;假不了就不顯示
- [ ] 顯示數字走真源(SSOT/API),非硬編碼
- [ ] 改/刪前 grep 確認引用;版面/文案零意外變動

**一句話:單一真相源、零捏造、顯示走真源、根因修。**
